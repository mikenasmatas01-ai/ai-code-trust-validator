## 🔍 What this app does

ai-code-trust-validator helps you check code made by AI before you use it. It looks for security risks, logic mistakes, and places where AI may have made things up. It is made for people who want a simple way to review AI-generated code without reading every line by hand.

Use it when you want to:
- Check AI code for weak spots
- Spot code that looks wrong or incomplete
- Review files before you copy them into your project
- Reduce the chance of bugs and unsafe code

## 🧭 How to use it

Once the app opens, you can use it to review code in a few simple steps:

1. Paste or load the AI-generated code
2. Choose the checks you want to run
3. Start the scan
4. Read the results
5. Fix the issues the app points out

The app is built to help with three main areas:

- **Security**: checks for risky code paths, unsafe input use, and weak safeguards
- **Hallucinations**: finds code that looks invented, misplaced, or out of line with the rest of the file
- **Logic errors**: spots code that may not work the way it should

## 🛠️ Features

### 🔐 Security checks
The app scans for common code risks, such as:
- Unsafe file use
- Weak input handling
- Missing checks before actions
- Code that may expose data

### 🧠 Hallucination checks
AI tools can invent names, functions, or patterns that do not fit the rest of the code. This app helps flag:
- Made-up function calls
- Fake library use
- Code that does not match the project style
- Odd claims in comments or logic

### ⚙️ Logic checks
It also helps find code that may fail in practice:
- Broken conditions
- Wrong order of steps
- Missing return paths
- Conflicting rules in the same file

### 📄 Easy-to-read results
Results are written in plain language so you can understand:
- What the issue is
- Where it appears
- Why it matters
- What to review next

## 🔎 What the checks look for

The app focuses on common problems found in AI-generated code:

- Hard-coded secrets or private values
- Unsafe file and path use
- Logic that skips a step or repeats a step
- Functions that do not match the rest of the code
- Text that sounds correct but does not fit the code flow
- Missing input checks
- Code blocks that may break under normal use

## 🧩 Best results

To get clearer results:
- Check one file or one small set of files at a time
- Use code that is already saved in plain text form
- Review the results before you copy the code elsewhere
- Fix the top issues first
- Re-run the check after changes
